## December 2025

Here are the highligths of the most important updates we added in Q4 2025.

### 1. Self-serve analytics with Custom Canvas

With Custom Canvas, end users can create their own charts using templates can be defined in the Embeddable builder. This lets them build dashboards independently while preserving the look and feel of your product, just like any other Embeddable dashboard.

### 2. Caching API per Enviornment 

The Caching API now allows security contexts to be managed separately for dev, staging, and prod environments, preventing updates in one environment from affecting the others. New environment-specific endpoints have been added while keeping the existing API compatible.

## 4. Preview of each component in component picker

In the Builder experience we now show a preview of each component in the component picker instead of just a list. This makes it clearer to see what component you are selecting to build with.

### 4. Support casting from one native type to another

This feature lets multiple components share and control the same variable, even if they expect different types or single vs. multiple values. It makes interactions like dropdowns and charts work together more smoothly without extra configuration.

### 5. Optimised chart configurator 

We now only show sub-inputs that are required by default, making it easier to use the chart configurator.

### 3. Data Access Policies

Access Policies make enforcing row-level security powerful, flexible, and safe. Reusable and applied at runtime, they support nested filters, role-based logic, and automatically guard against SQL injection.

With Access Policies, you can:
- Assign multiple roles per user without complex SQL workarounds
- Apply conditional rules (e.g., only for admin users) easily
- Build complex nested filters (AND/OR combinations) in a maintainable way
- Automatically handle user-provided values safely, protecting against SQL injection

We continue to support traditional SQL-based RLS, but Access Policies make it easier to manage security at scale.

Learn more about `Data Access Policies` [here](/data-modeling/access-policies).

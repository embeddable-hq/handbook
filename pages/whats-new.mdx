### 3. Data Access Policies

Control exactly what data each user can see based on their assigned roles. Define access policies that automatically apply at runtime, supporting:

- Multiple roles per user
- Conditional rules (e.g., only if a user is an admin)
- Complex nested filters (AND/OR combinations)
- Safe handling of user-provided values (SQL injection protection)

Learn more about `Data Access Policies` [here](/data-modeling/access-policies).

You can now test and run Embeddables against different Cube deployments (internal or external).

- Define and manage external data providers in your workspace
- Refresh metadata per environment directly in the builder or data playground
- Environment-aware UI for external Cube users, hiding internal-only features

Learn more about `Cube deployments` [here](/).


### 5. Timezone support 

You can now set a specific timezone for Embeddable dashboards.

This ensures all users see data in a consistent â€œreporting timezone,â€ avoiding confusion from browser-based timezones and supporting accurate aggregations across global teams.

- Pass a standard timezone string in clientContext and loadData for timezone-aware queries
- Schedule Cube pre-aggregations for specific timezones via the Caching API
- Default timezone remains UTC for backwards compatibility and omitted values
- Handles timezones globally, including daylight saving changes, across US, Asia, and London

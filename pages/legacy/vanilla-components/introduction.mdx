# Vanilla Components

<Callout emoji="‼️">
  Vanilla Components are now deprecated in favor of [Remarkable Pro](/component-libraries/remarkable-pro). We recommend using Remarkable Pro for all new projects, but if you have an existing implementation with Vanilla Components, you can continue to use and maintain it as needed.
</Callout>

Initially created as an example of what clients _could_ build with Embeddable, Vanilla Components evolved into a robust, production-ready component library. It offered a solid foundation for teams looking to create custom analytics experiences with speed and flexibility.

**Vanilla Components are now being sunset.** We took the lessons we learned from this project and built [Remarkable Pro](/component-libraries/remarkable-pro/introduction), but as many teams are still using Vanilla Components, we've kept the documentation available.

Vanilla Components includes:

- **Core visualizations**: bar, line, pie/donut, KPI tiles, tables, heatmaps, etc.
- **Essential controls**: dropdowns, multi-selects, date pickers (UTC-safe and time zone friendly), and custom filters.
- **Theming support**: easily customize colors, typography, and spacing to match your brand.
- **extensibility**: override any component with a custom implementation.

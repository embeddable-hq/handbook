Hello everyone,

Today we‚Äôre going to talk about creating a custom component in Embeddable. We‚Äôve covered this in the docs, of course, but this will allow us to simultaneously build something a bit more complex and cover things in a simple, step-by-step fashion.

To that end, we‚Äôre going to build a component using Chart.js that doesn‚Äôt exist in the Vanilla Components. Specifically, were going to build a Polar Area Chart. It‚Äôs similar to a pie chart, but it helps give additional context by scaling the wedges. We‚Äôre going to use Embeddable‚Äôs sample data for this, and we‚Äôre going to organize the chart by a dimension, then break it into slices by a measure.

One last thing to mention: just because we use Chart.js at Embeddable doesn‚Äôt mean you have to! We have clients using AmCharts, High Charts, and a variety of other options. If you can build it in React, you can build it in Embeddable.

We‚Äôll need two files to do this, which are key to the way that Embeddable works. The first is the `emb.ts` file, which controls what our chart inputs look like in the builder, how we load data, and how we pass it to the chart. The second is our `index.tsx` file, which is a pure React component. This is where we‚Äôll put all the actual chart code. Everything from from the `emb.ts` file will be passed to `index.ts` as props, just the way you‚Äôre used to with ordinary React.

## Creating the emb.ts File

As you can see, that‚Äôs an extremely basic React component. It‚Äôll grow as we build things out, but for now we just need something to reference in our `emb.ts` file. Let‚Äôs create `PolarAreaChart.emb.ts` and get started filling it out with the following code:

```tsx
import { OrderBy, loadData } from '@embeddable.com/core';
import {
  EmbeddedComponentMeta,
  Inputs,
  defineComponent,
} from '@embeddable.com/react';

import Component from './index';

export const meta = {
  name: 'PolarAreaChart',
  label: 'Polar Area Chart',
  classNames: ['inside-card'],
  category: 'Charts: essentials',
  inputs: [],
} as const satisfies EmbeddedComponentMeta;

export default defineComponent(Component, meta, {
  props: (inputs: Inputs<typeof meta>) => {
    return {
      ...inputs,
    };
  },
});
```

Not much here, yet, either. That‚Äôs okay! That makes it easier to understand what we‚Äôre dealing with. The first thing to notice is that we‚Äôre importing our Component from `index.tsx`. We pass that into the `defineComponent` function, which allows Embeddable to work its magic. After that, we define some meta information. Most of this is straightforward and just allows us to name and categorize the chart. That empty `inputs` array is important, though, and we‚Äôll be adding things to it shortly.

After that, we run `defineComponent`, which at the moment isn‚Äôt doing much. It‚Äôs just passing each of our inputs to the component as props ‚Ä¶ except we don‚Äôt have any inputs yet, so it‚Äôs passing an empty props object. That‚Äôs not very useful, so let‚Äôs add some inputs!

## Defining Inputs

First thing, we need a dataset input in order to wire your chart up to one of your datasets, so within the `inputs` array, let‚Äôs add this:

```tsx
    {
      name: 'ds',
      type: 'dataset',
      label: 'Dataset',
      description: 'Dataset',
      defaultValue: false,
      category: 'Chart data',
    },
```

We‚Äôre also going to need a dimension with which to slice up our pie:

```tsx
    {
      name: 'slice',
      type: 'dimension',
      label: 'Slice',
      config: {
        dataset: 'ds',
      },
      category: 'Chart data',
    },
```

And a measure for the size of the slices:

```tsx
    {
      name: 'metric',
      type: 'measure',
      label: 'Metric',
      config: {
        dataset: 'ds',
      },
      category: 'Chart data',
    },
```

This is technically all we need. We could add a lot more (and you‚Äôve probably noticed that most of the Vanilla Components have a lot more inputs in the builder). Let‚Äôs keep it simple for now, but we‚Äôre going to add two more inputs just to match the other components: a title and a description. Here they are:

```tsx
    {
      name: 'title',
      type: 'string',
      label: 'Title',
      description: 'The title for the chart',
      category: 'Chart settings',
    },
    {
      name: 'description',
      type: 'string',
      label: 'Description',
      description: 'The description for the chart',
      category: 'Chart settings',
    },
```

Before we switch to working with these inputs in `index.ts` we need to modify `defineComponent` to actually pull data from our data source. We do this with Embeddable‚Äôs built-in `loadData` function. Change the `return` statement from this:

```tsx
    return {
      ...inputs,
    };
```

to this:

```tsx
    return {
      ...inputs,
      results: loadData({
        from: inputs.ds,
        select: [inputs.slice, inputs.metric],
      }),
    };
```

## Building the Polar Area Chart

Great! We‚Äôve got everything we need to make the Polar Area Chart do something. Let‚Äôs get that working. Switch back to your code editor, and in `index.tsx`, add the following imports:

```tsx
import { useEffect, useState } from 'react';
import {
  CategoryScale,
  Chart as ChartJS,
  ChartData,
  Filler,
  Legend,
  LinearScale,
  PointElement,
  TimeScale,
} from 'chart.js';
import 'chart.js/auto';
import 'chartjs-adapter-date-fns';
import { PolarArea } from 'react-chartjs-2';
```

That‚Äôs everything we‚Äôll need to make things work. Next, let‚Äôs register a few defaults with Chart.js like this:

```tsx
ChartJS.register(
  CategoryScale,
  LinearScale,
  PointElement,
  Filler,
  Legend,
  TimeScale,
);
```

And just below that, we‚Äôll define a quick type for our props:

```tsx
type Props = {
  ds: Dataset;
  slice: Dimension;
  metric: Measure;
  title?: string;
  description?: string;
  results?: any;
};
```

All right, with the housekeeping out of the way, let‚Äôs jump down to our actual function. Right now it looks like this:

```tsx
const PolarAreaChart: React.FC = (props) => {
  console.log(props);
  return <div>This will be a polar area chart.</div>;
};
```

But we‚Äôre going to expand it a bit. First, remove the `console.log`, and after that, add our Props type to `React.FC` like this:

```tsx
const PolarAreaChart: React.FC<Props> = (props) => {
```

then just below the function declaration, add this:

```tsx
  const [data, setData] = useState<ChartData<'polarArea'>>({
    labels: [],
    datasets: [],
  });
```

Now that we have some empty chart data in state, we‚Äôll use `useEffect` to populate it. This code snippet‚Äôs a little long, so I‚Äôve commented it to explain what‚Äôs going on, and we‚Äôll also step through it.

```tsx
  useEffect(() => {
    // Pull labels and values from the dimension and measure, respectively
    if (props.results?.data?.length > 0) {
      const labels = props.results.data.map(
        (item: any) => item[props.slice.name],
      );
      const values = props.results.data.map(
        (item: any) => item[props.metric.name],
      );

			// Set ChartJS data, including the colors for the slices
      setData({
        labels, // set above (our slices)
        datasets: [
          {
            label: props.title || 'Polar Area Chart',
            data: values, // set above (our counts)
            backgroundColor: [
              '#6778DE',
              '#FF997C',
              '#9EA4F4',
              '#B8B8D1',
              '#FF6B6C',
              '#FFC145',
              '#9DC7FF',
              '#FF805B',
            ],
            borderWidth: 0,
          },
        ],
      });
    }
  // Make sure to re-run the useEffect if any values change
  }, [props.results, props.slice.name, props.metric.name, props.title]);
```

What we‚Äôre doing here is straightforward. First we check and see if we have data. If and when we do, we get our list of labels and values from the dimension and measure values. Then we pass that data to Chart.js along with some color values, a border width, and a label. Chart.js has a LOT of other options you can configure, either by hardcoding them or by tying them into additional inputs (and thus prop values) that you create in the `emb.ts` file. You can get as complex as you want, within the bounds of whatever library you‚Äôre working with.

Okay, last but not least, we need to make the actual Polar Area Chart. Where we currently have a one-line `return` statement, instead use this code:

```tsx
  return (
    <div style={{ height: '100%', maxHeight: '100%' }}>
      {props.title && <strong>{props.title}</strong>}
      {props.description && <p>{props.description}</p>}
      <div style={{ height: '90%', maxHeight: '90%' }}>
        {props.results?.isLoading ? (
          <p>Loading data...</p>
        ) : (
          <PolarArea data={data} options={{}} height="100%" />
        )}
      </div>
    </div>
  );
```

As you can see, we show the title and description if they exist. We also check the results provided by `loadData` for an `isLoading` boolean and if it‚Äôs true, we show some loading text (you could replace this with a spinner, which is how our Vanilla Components do things). Once `isLoading` is no longer true, we show the Polar Area chart filled with the appropriate data, which we set up in the `useEffect` above.
